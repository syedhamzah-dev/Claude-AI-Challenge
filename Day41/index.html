MODULE 01
Foundations

What is Supervised Learning, and What Does a "Fit" Mean?

We start from the smallest possible model — a straight line through data — because every technique later in this tutorial is a variation on the same idea: define an error, then shrink it.

Supervised learning, precisely

Supervised learning means learning a mapping from inputs (features) to known outputs (labels) using example pairs. You show the algorithm many (input, correct answer) pairs, and it learns a general rule it can apply to new, unseen inputs. There are two flavors, split by what the output looks like:

RegressionClassification
Output typeContinuous numberCategory / class label
Example question"What will this house sell for?""Is this email spam or not?"
Typical metricMSE, RMSE, R²Accuracy, Precision, Recall, F1
Analogy
Think of supervised learning like a new intern shadowing a senior analyst. The intern watches many (situation → decision) pairs. Over time they stop needing the senior analyst and start predicting the decision themselves — that's the trained model.

Simple linear regression

Simple linear regression fits a straight line ŷ = mx + b through your data, where m is the slope and b is the intercept. "Fitting" means choosing m and b so the line sits as close as possible to every point.

Worked example
Predicting exam score from hours studied. Say the true relationship is roughly score ≈ 8×hours + 40. A student who studies 5 hours might score around 80. Linear regression's job is to discover that "8" and "40" purely from example (hours, score) pairs — it never sees the formula, only the dots.

The cost function: measuring "how wrong"

For every point, the residual is the vertical gap between the actual value and the line's prediction: residual = y − ŷ. We square each residual (so positive and negative errors don't cancel, and big misses are punished more) and average them — that's Mean Squared Error (MSE):

MSE = (1/n) · Σ (yᵢ − ŷᵢ)²
n = number of points · yᵢ = actual value · ŷᵢ = predicted value on the line

Gradient descent is the algorithm that nudges m and b a small step at a time, always in the direction that reduces MSE fastest — like walking downhill on an "error landscape" with your eyes closed, feeling for the steepest slope under your feet.

Dashed lines = residuals. The best-fit line minimizes the sum of squared residual lengths — not the sum of raw residuals.

Common misconception
"A good fit means the line touches as many points as possible." False — the best-fit line usually touches almost none of the points. It minimizes total squared distance across all points, which is a global balance, not a local one. Also: correlation in the data does not prove causation — hours studied and score both being high could share a hidden cause (like motivation).
Key takeaways
  • Supervised learning = learning input→output mapping from labeled examples
  • Regression predicts numbers; classification predicts categories
  • A line is defined by slope (m) and intercept (b); "fitting" = choosing them to minimize error
  • MSE squares residuals so errors can't cancel out and large misses are penalized more
  • Gradient descent walks downhill on the error surface to find the minimum-error m, b
MODULE 02
Core Algorithms

From Lines to Decisions: Logistic Regression & Decision Trees

Real problems rarely have one input or a numeric answer. Here we extend to multiple features and pivot from predicting numbers to predicting categories.

Multiple linear regression

Add more features and the line becomes a plane, then a hyperplane: ŷ = w₁x₁ + w₂x₂ + ... + wₙxₙ + b. Each weight wᵢ tells you how much ŷ moves per unit change in that feature, holding the others constant — e.g. predicting house price from square footage, bedrooms, and age together instead of one at a time.

Logistic regression: from numbers to probabilities

To classify (e.g. spam / not spam), we still start with a linear combination z = w·x + b, but squash it through the sigmoid function so the output is always between 0 and 1 — a probability.

σ(z) = 1 / (1 + e⁻ᶻ). Large positive z → probability near 1. Large negative z → near 0.

A decision boundary is drawn wherever that probability crosses your chosen threshold (commonly 0.5) — everything on one side is predicted class A, the other side class B.

Analogy
Logistic regression is a dimmer switch, not an on/off switch. It doesn't just say "spam" — it says "87% sure this is spam," and you decide where to draw the line for action.

Decision trees

A decision tree classifies by asking a sequence of yes/no questions about your features, splitting the data at each step to make the resulting groups as "pure" (single-class) as possible. Purity is measured with Gini impurity or entropy — both are just different ways of scoring "how mixed is this group of labels?"

Worked example
Predicting churn (yes/no) from tenure, monthly charges, and contract type. A tree might first split on contract type (month-to-month vs. yearly), then split each branch again on tenure — mirroring how a human analyst would reason through the same decision manually.

Common misconception
"Logistic regression is a regression technique for predicting numbers." It has "regression" in the name for historical reasons (it regresses the log-odds), but it is used for classification, not for predicting arbitrary continuous values.
Key takeaways
  • Multiple linear regression extends a line to many features via weighted sums
  • Logistic regression outputs a probability via the sigmoid function, then applies a threshold
  • Decision trees split data repeatedly to maximize the purity of resulting groups
  • All three are interpretable: you can read off exactly why a prediction was made
MODULE 03
Practical Application

Making Models That Generalize: Overfitting & Validation

A model that scores perfectly on the data it was trained on can still fail badly on new data. This module is about catching that before it costs you in production.

Train/test split

Never evaluate a model on the same data it learned from — that's like grading a student using the exact questions they memorized answers to. Instead, split your dataset (commonly 70–80% train, 20–30% test) before training, and only touch the test set once, at the end, to estimate real-world performance.

Analogy
Training data is the practice exam with an answer key. Test data is the real exam, seen for the first time. A model that only does well on the practice exam hasn't actually learned the subject — it's memorized the answer key.

Overfitting vs. underfitting

Regularization & cross-validation, briefly

Regularization (L1/Lasso and L2/Ridge) adds a penalty for large weights to the cost function, discouraging the model from relying too heavily on any single feature — a built-in pressure toward simplicity. Cross-validation (e.g. k-fold) repeats the train/test split several times over different slices of data, then averages the results, giving a more reliable performance estimate than one lucky or unlucky split.

Worked example
A churn model that hits 99% accuracy on training data but only 68% on new customers is almost certainly overfit. Trying L2 regularization or pruning the decision tree's depth usually closes most of that gap.
Common misconception
"Higher training accuracy always means a better model." Training accuracy alone tells you nothing about generalization — it can rise as the model memorizes noise while real-world performance quietly gets worse.
Key takeaways
  • Always evaluate on held-out data the model never trained on
  • Underfitting = too simple, high error everywhere; overfitting = too complex, low train error but high test error
  • Regularization penalizes complexity directly inside the cost function
  • Cross-validation gives a more trustworthy performance estimate than a single split

Choosing the Right Metric: Evaluation That Matches the Problem

A model is only as good as the metric you judge it by. This module turns raw predictions into the numbers that actually drive decisions.

The confusion matrix

Precision, recall, F1, accuracy

MetricFormulaAnswers

F1 Score2·(P·R)/(P+R)Balance of precision and recall
Analogy
A smoke detector with high recall rarely misses a real fire but may false-alarm on burnt toast (low precision). A smoke detector with high precision only rings for real fires but might stay silent during a small one it's unsure about. Neither is "correct" in isolation — it depends what a missed alarm costs you versus a false one.

ROC-AUC, and metrics for regression

ROC curve plots True Positive Rate vs. False Positive Rate across every possible threshold. AUC (area under curve) close to 1.0 = excellent separation between classes; 0.5 = no better than a coin flip.

For regression problems, use MSE or RMSE (same units as your target — easier to interpret) for average error size, MAE when you want to be less sensitive to outliers, and R² to answer "what fraction of the variance in the target does my model explain?" (0 to 1, higher is better).

Worked example
For a cancer screening classifier, missing a real case (FN) is far more costly than a false alarm (FP) — so you'd optimize for high recall even if precision drops. For a spam filter, a false positive (blocking a real email) is often worse than missing some spam — so precision matters more there.
Common misconception
"The model with the highest accuracy is the best model." On imbalanced data (e.g. 98% non-fraud, 2% fraud) a model that always predicts "not fraud" scores 98% accuracy while being completely useless. Always check precision/recall on the minority class too.
Key takeaways
  • Confusion matrix (TP, FP, FN, TN) is the foundation every classification metric is built from
  • Precision guards against false alarms; recall guards against missed positives — pick based on real-world cost
  • Accuracy is misleading on imbalanced datasets
  • ROC-AUC summarizes performance across all thresholds at once
  • For regression, prefer RMSE/MAE for error size and R² for explained variance

Summary notes

Supervised learning maps inputs to known outputs. Regression predicts numbers by minimizing squared error (MSE) via gradient descent; classification predicts categories, most simply via logistic regression's sigmoid-squashed probability or a decision tree's sequence of splits. A model is only useful if it generalizes — check this with a train/test split, watch for the overfitting gap, and use regularization or cross-validation to control complexity. Finally, judge classifiers with the metric that matches the real cost of mistakes: precision when false alarms are expensive, recall when missed positives are expensive, F1 for a balance, and ROC-AUC for a threshold-independent summary; judge regressors with RMSE/MAE for error size and R² for explained variance.
