# Day 3 – Role-Based Prompting

## Objective

To understand how assigning different roles (personas) to an AI changes the quality, focus, and perspective of its responses.

---

## Question Used

**What should I do this weekend to improve my career?**

---

# Prompt 1: General Advice (No Role)

## Prompt

```
What should I do this weekend to improve my career?
```

## Key Points from Response

- Build something tangible
- Learn a high-leverage skill
- Write publicly
- Have one real conversation
- Do a career audit
- Update resume, LinkedIn, and portfolio

## Observation

The response provided broad career advice applicable to almost anyone. It focused on personal growth, networking, learning, and maintaining professional profiles.

---

# Prompt 2: Startup Founder Persona

## Prompt

```
You are a successful startup founder.

What should I do this weekend to improve my career?
```

## Key Points from Response

- Stop consuming and start shipping
- Talk to potential customers
- Find your biggest bottleneck
- Share your thinking publicly
- Connect with someone ahead of you

## Observation

The response focused on entrepreneurship, action-taking, networking, customer discovery, and building in public. The advice was centered around creating opportunities and solving real-world problems.

---

# Prompt 3: Senior Software Developer Persona

## Prompt

```
You are a senior software developer.

What should I do this weekend to improve my career?
```

## Key Points from Response

- Go deeper into one technology
- Build under constraints
- Read source code instead of only documentation
- Write technical decision documents
- Review past code
- Contribute meaningful open-source code

## Observation

The response focused heavily on technical growth, engineering practices, code quality, and software craftsmanship. It encouraged developing deeper technical expertise.

---

# Comparison of Responses

| Aspect | General Advice | Startup Founder | Senior Software Developer |
|----------|----------|----------|----------|
| Focus | Career Growth | Business & Entrepreneurship | Technical Mastery |
| Main Goal | Personal Development | Create Opportunities | Improve Engineering Skills |
| Recommended Activities | Learning, Networking | Customer Discovery, Building | Coding, Open Source, Deep Learning |
| Perspective | Broad | Business-Oriented | Technical-Oriented |

---

# Key Learnings

1. The same question generated significantly different responses when a role was assigned.
2. Role-Based Prompting helps AI respond from a specific perspective.
3. Founder personas prioritize business, customers, and execution.
4. Developer personas prioritize technical depth, code quality, and engineering skills.
5. Without a role, responses are generally broader and less specialized.

# Conclusion

This exercise demonstrated the power of Role-Based Prompting. By assigning a specific role, AI responses become more focused, relevant, and aligned with the mindset of that profession. This technique is useful for obtaining specialized advice and exploring different perspectives on the same problem.
